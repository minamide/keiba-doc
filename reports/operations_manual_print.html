1. 日次レポート生成フロー

1.1 基本的な使い方

💡 最も簡単な方法
# 指定日付のレポート生成（HTML形式）
python scripts/gen_daily_report.py 20260328

オプション指定：

# フォーマット指定
python scripts/gen_daily_report.py 20260328 --format html  # HTML (デフォルト)
python scripts/gen_daily_report.py 20260328 --format json  # JSON

# Kura CMS へアップロード
python scripts/gen_daily_report.py 20260328 --upload

1.2 出力ファイル

reports/ ├── daily_report_20260328.html # HTML形式（ブラウザで表示） └── daily_report_20260328.json # JSON形式（API用）

1.3 レポートに含まれる情報

  • 推奨馬券テーブル
    • レース情報（日時、場所、R数）
    • 推奨馬番・馬名
    • 券種（単勝・複勝・馬連・ワイド・馬単）
    • EV（期待値）・オッズ・投資額
  • グラフ・統計
    • 券種別推奨件数
    • EV分布
    • 投資額分布
  • 出馬表セクション
    • 各レースの出馬情報
    • 推奨馬のハイライト

2. データ取得（スクレイピング）

2.1 出馬表データの取得

時期 前日〜開催当日（レース前日に実行推奨）

# 指定日付のレースIDを自動検出して出馬データを取得
node scripts/scraping/scrapeShutuba.js --year 2026 --month 3

# または特定の日付を指定
node scripts/scraping/scrapeShutuba.js --date 20260328

# 特定のレースIDを直接指定（テスト用）
node scripts/scraping/scrapeShutuba.js --id 202606030101 202606030102

出力: data/external/shutuba_html/ に HTML キャッシュ、DB に登録

2.2 JRA公式オッズの取得 重要

時期 レース開催日の発走直前（重要！）

⚠️ オッズなしで予想できません

オッズデータが存在しないと、EV = 0 になり、推奨馬券が出ません。

必ず開催当日に JRA オッズを取得してください。

JRA公式オッズは取得可能なレースを自動検出します：

# 利用可能なレース一覧を確認
node scripts/scraping/scrapeJraOdds.js --list

# 全レースのオッズを取得
node scripts/scraping/scrapeJraOdds.js

# 特定のレースのみ取得
node scripts/scraping/scrapeJraOdds.js --race-id 202606030101 202606030102 202606030103

# リクエスト間隔を調整（デフォルト800ms）
node scripts/scraping/scrapeJraOdds.js --delay 1000

出力: data/external/scraped/netkeiba.db の odds_win テーブルに登録

ℹ️ 注意事項
  • 複数回実行可: 更新されたオッズが DB に反映されます
  • 前日取得NG: 前日のオッズは確定していません
  • 開催直前推奨: 15時頃のオッズが最終確定です

2.3 NetKeiba オッズ API の取得（参考）

確定オッズ確認用：

node scripts/scraping/scrapeNetkeibaOdds.js --date 20260328

4. トラブルシューティング

4.1 「馬が一頭も表示されない」

⚠️ 原因: オッズデータが存在しない

オッズなし → EV = 0 → 推奨馬券なし → 馬が表示されない

解決方法:

# JRAオッズを取得
node scripts/scraping/scrapeJraOdds.js

# レポート再生成
python scripts/gen_daily_report.py 20260328

4.2 「オッズが取得できない」

原因: JRA サイトが応答していない、または開催がない

対策:

  • リトライを増やす: --retries 5
  • リクエスト間隔を広げる: --delay 2000
  • 時間をおいて再実行

4.3 「特徴量が空です」エラー

解決方法:

node scripts/scraping/scrapeShutuba.js --date 20260328
python scripts/gen_daily_report.py 20260328

4.4 「推奨なし（EV閾値未達）」

✓ 正常な状況です

毎日推奨があるとは限りません。複数日の累積データで判断してください。

9. 注意事項

⚠️ 重要なポイント

1. レースIDは日付ではない

  • 202606030101 = 中山1開催1R（開催コード）
  • kaisai_date と race_id は別物

2. オッズなしで予想できない

  • オッズがないと EV = 0
  • 必ず開催当日にオッズを取得

3. 馬名の重複対策

  • 複数ソースで馬名が異なる場合あり
  • 最新データが優先される

4. データベースは累積型

  • 定期的にバックアップしてください
